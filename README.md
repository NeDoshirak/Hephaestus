# Hephaestus

![Hephaestus Logo](frontend/public/logo.svg)

## Что это?

**Hephaestus** — сервис для анализа и управления данными о вакансиях. Система автоматически собирает вакансии с HeadHunter, извлекает требуемые навыки с помощью AI, нормализует данные и предоставляет аналитику по рынку труда.

### Основные возможности

- **Поиск вакансий** — интеграция с HeadHunter API
- **Анализ требований** — AI извлекает навыки и компетенции из описаний вакансий
- **Нормализация данных** — система унифицирует названия навыков (например, "JavaScript", "JS", "javascript" → одно каноническое имя)
- **Аналитика** — агрегация данных по месяцам и профессиям для выявления тенденций
- **Модерация** — контроль качества новых навыков перед добавлением в справочник

## Как это работает

### Три слоя данных

1. **Сырые данные (Raw Layer)**
   - Вакансии и описания в исходном виде с HeadHunter
   - Автоматически извлеченные навыки (еще не проверены)
   - Хранятся в PostgreSQL

2. **Справочник (Reference Layer)**
   - Проверенные и нормализованные навыки
   - Синонимы и связи между навыками (например, "JavaScript" и "JS" — это один навык)
   - Каноническое имя каждого навыка
   - Хранятся в PostgreSQL

3. **Аналитика (Analytics Layer)**
   - Статистика: какие навыки чаще всего требуются
   - Тренды: как менялись требования по месяцам
   - Профили: какие навыки идут в паре (например, React часто требуется вместе с JavaScript)
   - Хранятся в ClickHouse для быстрых запросов

### Поток данных

```
HeadHunter API
      ↓
[Сборка вакансий]
      ↓
PostgreSQL (сырые данные)
      ↓
[AI анализ: извлечение навыков]
      ↓
PostgreSQL (требует модерации)
      ↓
[Модератор проверяет новые навыки] ← Вы здесь управляете
      ↓
PostgreSQL (справочник)
      ↓
[Агрегация статистики]
      ↓
ClickHouse (аналитика)
```

### Модерация и управление

Когда система извлекает новый навык из вакансии (например, "React.js"), он сначала появляется как неповеренный. Модератор:

1. Просматривает новый навык
2. Решает: это новый навык или это синоним существующего?
3. Если новый — добавляет в справочник
4. Если синоним — связывает с существующим навыком

Это нужно, чтобы избежать дублей (система не считала бы "JavaScript" и "JS" разными навыками).

## Быстрый старт

### Требования

- **Docker** и **Docker Compose** (или запуск локально)
- Для локального запуска: **Node.js 16+**, **.NET 10+**, **PostgreSQL**

### Запуск с Docker Compose

**Перед первым запуском:** настрой секреты для HeadHunter

```bash
# 1. Скопируй пример конфига
cp .env.example .env

# 2. Открой .env и заполни свои секреты:
#    HEADHUNTER_ACCESS_TOKEN=...
#    HEADHUNTER_CLIENT_SECRET=...
# 
#    Если у тебя уже есть секреты в dotnet user-secrets, используй их оттуда:
#    cd src/Hephaestus && dotnet user-secrets list
```

Затем запусти все сервисы:

```bash
cd Hephaestus
docker compose up
```

Откроется:
- **Frontend** (админка): http://localhost:3001
- **Backend API**: https://localhost:7168
- **PostgreSQL**: localhost:5432

### Первый запуск

1. Перейди на http://localhost:3001
2. Открой вкладку "Вакансии" → увидишь список вакансий в БД
3. Открой вкладку "Загрузить с HH" → введи поисковый запрос (например, "React") и загрузи вакансии
4. Система автоматически:
   - Скачает вакансии с HeadHunter
   - Извлечет требуемые навыки
   - Добавит в БД

### Для разработки (локально)

**Backend:**
```bash
cd src

dotnet run --project Hephaestus/Hephaestus.csproj
```

## Три главные страницы админки

### 1. "Вакансии"

Управление вакансиями в БД:
- Просмотр списка (с пагинацией и поиском)
- **Фильтр по ключевым навыкам** — выбери навык из выпадающего списка, чтобы увидеть только вакансии с этим требованием
- Добавление новой вакансии
- Редактирование описания и навыков
- Удаление вакансий

### 2. "Загрузить с HH"

Импорт вакансий с HeadHunter:
- Ввод поискового запроса (например, "Senior React Developer")
- Одна кнопка "Загрузить" — система сама:
  - Скачает вакансии
  - Извлечет навыки
  - Добавит в БД
- Статус загрузки (сколько вакансий добавлено)

### 3. "Поиск на HH"

Прямой поиск по HeadHunter:
- Введи поисковый запрос
- Просмотри результаты с HeadHunter
- Открой вакансию на сайте HH прямой ссылкой

## Оптимизация и кешивание

Система оптимизирована для снижения нагрузки:

- **Список вакансий** кешируется на 30 минут (не перезагружается каждый раз)
- **Результаты поиска HH** кешируются на 15 минут
- **Детали вакансии** кешируются на 1 час
- CRUD операции обновляют только изменяемые данные, не перезагружают весь список

## Решение проблем

### Страница не загружается

**Проблема:** "Ошибка сервера" или белый экран

**Решение:**
1. Проверь что все сервисы запустились: `docker compose ps`
2. Посмотри логи: `docker compose logs`
3. Перезагрузи страницу (Ctrl+R или Cmd+R)

### Вакансии не загружаются

**Проблема:** Список вакансий пустой

**Решение:**
1. Открой вкладку "Загрузить с HH"
2. Введи запрос (например, "Python")
3. Нажми "Загрузить"
4. Вернись на "Вакансии" — должны появиться новые

### Ошибка подключения к HeadHunter

**Проблема:** "Error loading HH data"

**Решение:**
- HeadHunter API может быть временно недоступен
- Попробуй позже или используй другой поисковый запрос

## Недавние улучшения

### Поиск на HeadHunter (оптимизирован)
- **Клики работают**: Вакансии в результатах поиска правильно кликаются и показывают детали
- **Поиск по кнопке**: Ввод текста больше не вызывает автоматический поиск. Запрос отправляется только при нажатии кнопки или Enter
- **Правильная пагинация**: Переключение страниц работает корректно

### Фильтр по навыкам (с поиском)
- На странице "Вакансии" можно фильтровать список по ключевым навыкам
- **Поиск в фильтре**: Выпадающее меню имеет встроенный поиск для быстрого нахождения нужного навыка
- Не нужно скроллить через весь список навыков — просто введи первые буквы
- При выборе навыка показываются только вакансии, требующие этого навыка
