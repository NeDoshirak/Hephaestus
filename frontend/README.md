# Hephaestus Frontend

Админка на React для управления вакансиями и навыками. Управление вакансиями, импорт с HH, верификация и классификация навыков.

## О проекте

### Архитектура
- **React 18** + **TypeScript** + **Vite** для быстрой разработки
- **React Router** для навигации между вкладками
- **React Query** для кеширования и управления данными (оптимизировано: 30 мин стейлтайм)
- **Tailwind CSS** для стилей
- **Axios** для API запросов
- **lucide-react** для иконок

### Основные страницы
1. **Вакансии** - CRUD операции (список, создание, редактирование, удаление) с пагинацией и поиском
2. **Загрузить с HH** - импорт вакансий с HeadHunter по поисковому запросу
3. **Поиск на HH** - прямой поиск и просмотр деталей вакансий с HH API
4. **Верификация навыков** - проверка и одобрение навыков, извлечённых из вакансий
5. **Верифицированные навыки** - классифицированные, одобренные навыки с синонимами и зависимостями

### Цветовая схема
- Основной: #5CBBF1 (голубой)
- Тёмный: #313642 (для sidebar)
- Белый: #FFFFFF

### Кеширование (оптимизировано)
- Список вакансий: 30 минут
- Результаты поиска HH: 15 минут  
- Детали вакансии: 1 час
- CRUD операции обновляют только нужные данные в кеше (не перезагружают всё)
- Retry логика: 2 повтора при ошибках

## Быстрый старт

### Требования
- Node.js 16+

## Как расширять

### Добавить новую страницу
1. Создать компонент в `src/pages/NewPage.tsx`
2. Добавить маршрут в `App.tsx`
3. Добавить вкладку в `Sidebar.tsx` (если нужна)

### Добавить новый API запрос
1. Добавить метод в `src/services/api.ts`
2. Создать хук в `src/hooks/` если часто используется
3. Использовать React Query с правильным стейлтайм

### Изменить цвета
- Обновить `tailwind.config.js` (extends.colors)
- Или изменить классы Tailwind в компонентах (#5CBBF1, #313642)

## Управление навыками

### Верификация навыков (SkillsReviewPage)
Процесс верификации навыков, извлечённых из вакансий:

1. **Загрузка навыков** - импорт сырых навыков из вакансий на бэкенде
2. **Проверка** - пользователь видит список навыков на проверке с количеством упоминаний
3. **Три способа одобрения:**
   - **Создать новый** - создать новый независимый навык в системе
   - **Добавить как синоним** - добавить к существующему навыку (например, "С#" как синоним к "C#")
   - **С зависимостями** - создать новый навык и связать его с существующим (parent-child отношение)

**UI:** Клик на навык раскрывает форму одобрения вниз (inline expandable), где можно выбрать способ, ввести название, описание и зависимости.

### Верифицированные навыки (CleanSkillsPage)
Список полностью одобренных и классифицированных навыков:

- **Синонимы** - альтернативные названия для одного навыка (например, "csharp", "с#" → "C#")
- **Зависимые навыки** - что требует этот навык (например, WPF требует C#)
- **Родительские навыки** - на чём это основано (например, WPF зависит от C#)
- **Фильтрация:**
  - Все навыки
  - Только с зависимыми навыками
  - Только с родительскими навыками

**Счётчик** - сколько раз этот навык встречается в вакансиях.

## Важные детали для разработки

- **staleTime vs gcTime**: Первый - когда данные считаются "старыми", второй - сколько хранить в памяти
- **Мутации оптимизированы**: Create/Update/Delete не перезагружают весь список - обновляют кеш
- **HTML теги в описаниях**: Отображаются как полноценный HTML (не удаляются)
- **Пагинация**: Показывает максимум totalPages кнопок, не даёт перейти на несуществующую страницу

## API Endpoints (из бэкенда)

**Вакансии:**
- `GET /api/vacancy?page=1&pageSize=10&search=` - список
- `GET /api/vacancy/{id}` - одна вакансия
- `POST /api/vacancy` - создать
- `PUT /api/vacancy/{id}` - обновить
- `DELETE /api/vacancy/{id}` - удалить

**Импорт:**
- `GET /api/vacancy/save?search=React` - загрузить с HH

**Навыки (верификация):**
- `POST /api/skills/import-from-vacancies?vacancyNameFilter=C#&limit=100` - импорт навыков из вакансий
- `GET /api/skills/on-review` - навыки на проверке (pending)
- `POST /api/skills/on-review/{id}/approve` - одобрить навык
- `POST /api/skills/on-review/{id}/reject` - отклонить навык

**Навыки (верифицированные):**
- `GET /api/skills/clean` - список верифицированных навыков
- `POST /api/skills/clean/add` - добавить новый навык вручную

**Прямой доступ к HH API:**
- `GET /api/hh/vacancies?text=React&page=0&perPage=20` - поиск
- `GET /api/hh/vacancies/{vacancyId}` - детали

## Troubleshooting

- **CORS ошибка** → проверить что бэкенд запущен на правильном порту
- **Кнопка не работает** → проверить DevTools (Network/Console)
- **Данные не обновляются** → очистить кеш: откроется = перезагрузить страницу
